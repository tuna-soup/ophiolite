**Audience:** script, notebook, and automation builders  
**Status:** Preview

Workflow-building in Ophiolite currently spans two public surfaces:

- **Python** for project-oriented orchestration and typed results
- **CLI** for inspection, packaging, and repeatable command execution

That split is deliberate for now. The public wrappers are still widening, but they already share one runtime and one vocabulary.

## Recommended path

1. Start with [Python quickstart](/docs/build-workflows/python-quickstart/)
2. Read [stable public vocabulary](/docs/build-workflows/stable-public-vocabulary/)
3. Read [built-in operator workflows](/docs/build-workflows/built-in-operator-workflows/)
4. Read [operator catalog and composition](/docs/build-workflows/operator-catalog-and-composition/)
5. Add [CLI automation](/docs/build-workflows/cli-automation/) where repeatability matters
6. Use [reference](/reference/) for exact commands and payload shapes
